# MDink Solutions — ملخّص تحديثات الجلسة (Blog + CMS + Operations)

هذا الملف يوثّق التعديلات المنفّذة في جلسة الإصلاح الشاملة. اقرأه مع
`README_DEPLOY.md` لمعرفة ترتيب التشغيل.

---

## 1) المدونة (Blog) — نظام كامل

### المشكلة الجذرية التي أُصلحت
- **المقال لم يكن يفتح** لأن `blog.$slug.tsx` كان route فرعيًا تحت `blog.tsx`
  الذي لا يحتوي `<Outlet />`. الحل: أُعيد تسمية الملف إلى `blog_.$slug.tsx`
  (route مستقل)، مع بقاء الرابط العام `/blog/:slug` كما هو.
- **المقالات لم تكن مزروعة** في القاعدة (مقال منشور واحد فقط). الحل: سكربت زرع.

### الميزات الجديدة
- **نظام بلوكات المحتوى** (`content_blocks` jsonb): heading, paragraph, list,
  note, quote, image, video, cta. يُعرض عبر `BlockRenderer`.
- **الأسئلة الشائعة (FAQ)** في نهاية كل مقال عبر `FaqAccordion` + بيانات مهيكلة
  (FAQPage JSON-LD) للـSEO.
- **عدّاد المشاهدات** عبر دالة `increment_blog_views`.
- **محرر لوحة تحكم حقيقي**: إضافة/تعديل/حذف/ترتيب البلوكات، رفع صور،
  إضافة فيديو، تحرير الأسئلة والأجوبة (زيادة/تقليل).
- **تصنيفات ديناميكية**: إدارة كاملة (إضافة/تعديل/حذف/إخفاء/ترتيب) من تبويب
  "التصنيفات" داخل صفحة المدونة، وتنعكس فورًا على فلاتر المدونة العامة.
- **تحقّقات**: slug فريد، next_slug يجب أن يشير لمقال منشور، related_slugs
  يجب أن تكون موجودة.
- المؤلف الافتراضي: **فريق MDink Solutions**.

### الملفات
- كود: `src/components/BlockRenderer.tsx`, `FaqAccordion.tsx`,
  `src/components/admin/BlocksEditor.tsx`, `FaqEditor.tsx`, `CategoriesManager.tsx`.
- migration: `20260630090000_blog_blocks_and_faq.sql`.
- زرع: `scripts/parse-articles.mjs`, `scripts/seed-blog-articles.mjs`, `seed-articles/`.

## 2) نظام الفيديو الموحّد
- `src/lib/video.ts` + `src/components/VideoPlayer.tsx`: يفهم YouTube
  (عادي/قصير/Shorts)، Vimeo، وMP4. يعرض إطارًا أفقيًا 16:9 أو عموديًا
  (Shorts/موبايل) بصورة مصغّرة وزر تشغيل — لا iframe مكسور.
- مستخدم في: بلوكات المقال، فيديو قسم المنظومة بالصفحة الرئيسية، شهادات الفيديو.

## 3) الآراء (Reviews)
- استُبدل مشغّل الفيديو الخام المكسور بكروت احترافية تستخدم `VideoPlayer`.
- migration `20260630100000_testimonials_tables.sql` يضمن جدولي
  `video_testimonials` و`written_testimonials` (كانا مستخدَمين دون تعريف).
- العنوان: "آراء عملاء MDink Solutions".

## 4) CMS الصفحات
- **الرئيسية**: رفع صورة الهيرو (تُخفي كارت المعاينة القديم تلقائيًا)،
  وقسم "المنظومة الرقمية" أصبح يدعم فيديو يُدار من اللوحة.
- **التواصل**: migration `20260630110000_social_links.sql` يضبط ترتيب الروابط
  (واتساب ← LinkedIn ← إنستجرام).
- **من نحن**: قسم الكواليس لم يعد يعرض بطاقات فارغة — يظهر فقط عند وجود صور.
- **زر واتساب العائم**: أخضر، في كل الصفحات، يُدار من الإعدادات (كان جاهزًا).

## 5) العمليات — دمج الطلبات
- `admin.leads` أصبح **موديولًا موحّدًا** يجمع الطلبات من ثلاثة مصادر
  (`contact_submissions`, `doctor_applications`, `consultations`) في واجهة
  واحدة مع فلترة بالنوع/الحالة/البحث، تغيير حالة، حذف، وتصدير Excel.
- تغيير الحالة يُكتب في الجدول الأصلي الصحيح حسب مصدر الطلب (لا نقل بيانات).
- السايدبار: ثلاثة بنود دُمجت في بند واحد "طلبات العملاء / Leads".

## 6) الأمان
- `service_role` موجود فقط في `client.server.ts` (سيرفر) — لا تسرّب في الواجهة.
- إنشاء المستخدمين عبر Edge Function مع تحقّق صلاحية (401/403).

## ملاحظة توافق قاعدة البيانات
جميع migrations هذه الجلسة `IF NOT EXISTS` / `ADD COLUMN IF NOT EXISTS`،
وتتحقق من القيود بالأعمدة لا بالأسماء، فلا تتعارض مع `SUPABASE_SETUP.sql`
ولا تؤذي أي قاعدة قائمة.
